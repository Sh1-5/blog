# 项目优化建议

基于对此 Next.js 14 个人作品集/博客网站的全面分析，以下是按优先级排列的优化建议。

---

## 🔴 关键优先级（高影响，应首先修复）

### 1. 启用图片优化
**文件**: `next.config.mjs:8-9`

**问题**: 当前设置 `unoptimized: true` 禁用了所有 Next.js 图片优化功能

**影响**:
- 加载性能差
- 没有懒加载
- 没有响应式图片
- 所有图片（头像、关于页面肖像、博客文章图片）都绕过了压缩和优化

**修复方案**:
```javascript
// next.config.mjs
images: {
  unoptimized: false,  // 改为 false 或删除此行
  formats: ['image/avif', 'image/webp'],
  remotePatterns: [
    // 添加具体的域名限制，而不是通配符
  ]
}
```

### 2. 为博客文章添加静态生成
**文件**: `src/app/blogs/[slug]/page.tsx`

**问题**: 缺少 `generateStaticParams()` — 所有博客都是服务端渲染（SSR）

**影响**:
- 页面加载速度慢
- 不必要的服务器处理
- 无法利用静态站点生成的性能优势

**修复方案**:
```typescript
// src/app/blogs/[slug]/page.tsx
export async function generateStaticParams() {
  const blogs = await getAllBlogs()
  return blogs.map((blog) => ({
    slug: blog.slug,
  }))
}

// 添加重新验证时间
export const revalidate = 3600 // 每小时重新验证
```

### 3. 实现博客内容缓存
**文件**: `src/lib/blogs.ts:22`

**问题**: 每次请求都从文件系统读取所有 MDX 文件

**影响**:
- 随着文章增多，性能下降
- 响应时间变慢
- 服务器负载增加

**修复方案**:
```typescript
// src/lib/blogs.ts
let cachedBlogs: Blog[] | null = null
let cacheTime = 0
const CACHE_TTL = 1000 * 60 * 5 // 5分钟

export async function getAllBlogs(): Promise<Blog[]> {
  const now = Date.now()
  if (cachedBlogs && (now - cacheTime) < CACHE_TTL) {
    return cachedBlogs
  }

  const blogFiles = await glob('**/*.mdx', { cwd: BLOGS_DIR })
  // ... 现有逻辑

  cachedBlogs = blogs
  cacheTime = now
  return blogs
}
```

或者使用 Next.js 的 `unstable_cache`:
```typescript
import { unstable_cache } from 'next/cache'

export const getAllBlogs = unstable_cache(
  async () => {
    const blogFiles = await glob('**/*.mdx', { cwd: BLOGS_DIR })
    // ... 现有逻辑
  },
  ['all-blogs'],
  { revalidate: 3600 }
)
```

### 4. 优化分析服务加载
**文件**: `src/components/analytics/analytics.tsx:5-20`

**问题**: 三个分析服务（OpenPanel、Google Analytics、Plausible）无条件加载

**影响**:
- 不必要的网络请求
- JavaScript 开销增加
- 页面加载变慢

**修复方案**:
```typescript
// src/components/analytics/analytics.tsx
'use client'

import dynamic from 'next/dynamic'

const OpenPanelComponent = dynamic(() => import('@openpanel/nextjs').then(mod => mod.OpenPanelComponent), {
  ssr: false,
})
const GoogleAnalytics = dynamic(() => import('@next/third-parties/google').then(mod => mod.GoogleAnalytics), {
  ssr: false,
})
const PlausibleProvider = dynamic(() => import('next-plausible').then(mod => mod.default), {
  ssr: false,
})

export function Analytics({ children }: { children: React.ReactNode }) {
  const hasOpenPanel = process.env.NEXT_PUBLIC_OPENPANEL_CLIENT_ID
  const hasGA = process.env.NEXT_PUBLIC_GA_ID
  const hasPlausible = process.env.NEXT_PUBLIC_PLAUSIBLE_DOMAIN

  return (
    <>
      {hasOpenPanel && <OpenPanelComponent clientId={process.env.NEXT_PUBLIC_OPENPANEL_CLIENT_ID!} />}
      {hasGA && <GoogleAnalytics gaId={process.env.NEXT_PUBLIC_GA_ID!} />}
      {hasPlausible ? (
        <PlausibleProvider domain={process.env.NEXT_PUBLIC_PLAUSIBLE_DOMAIN!}>
          {children}
        </PlausibleProvider>
      ) : children}
    </>
  )
}
```

### 8. 合并重复的博客逻辑

**文件**: `src/app/feed/route.ts:37-46`

**问题**: RSS feed 重新实现了文件读取，而不是使用 `getAllBlogs()`

**影响**:
- 维护负担
- 行为可能不一致
- 代码重复

**修复方案**:
```typescript
// src/app/feed/route.ts
import { getAllBlogs } from '@/lib/blogs'

export async function GET() {
  const feed = new Feed({...})

  // 使用现有工具而不是重新实现
  const blogs = await getAllBlogs()

  blogs.forEach(blog => {
    feed.addItem({
      title: blog.title,
      id: `${siteUrl}/blogs/${blog.slug}`,
      link: `${siteUrl}/blogs/${blog.slug}`,
      description: blog.description,
      date: new Date(blog.date),
      author: [{ name: blog.author }],
    })
  })

  return new Response(feed.rss2(), {
    headers: { 'Content-Type': 'application/xml' },
  })
}
```

```typescript
async function getVisitStats() {
  const res = await fetch(`${process.env.NEXT_PUBLIC_SITE_URL}/api/visit-stats`, {
    next: { revalidate: 300 } // 5分钟缓存
  })
  return res.json()
}

export async function VisitData() {
  const data = await getVisitStats()

  return (
    <div className="flex items-center gap-4">
      <div>今日访问: {data.today}</div>
```

### 10. 为博客文章添加丰富的元数据

**文件**: `src/app/blogs/[slug]/page.tsx:24-26`

**问题**:
- Open Graph 元数据缺少 `publishedTime`、`author`、`image`
- 没有文章的 JSON-LD 结构化数据

**影响**:
- SEO 较差
- 社交媒体预览不够丰富

```typescript
// src/app/blogs/[slug]/page.tsx
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const blog = await getBlogBySlug(params.slug)
  if (!blog) return {}

  return {
    title: blog.title,
    description: blog.description,
    authors: [{ name: blog.author }],
    openGraph: {
      title: blog.title,
      description: blog.description,
      type: 'article',
      publishedTime: blog.date,
      authors: [blog.author],
      images: [
        {
          url: blog.image || '/og-image.png',
          width: 1200,
          height: 630,
        },
      ],
    },
    twitter: {
      card: 'summary_large_image',
      title: blog.title,
      description: blog.description,
      images: [blog.image || '/og-image.png'],
    },
  }
}

// 在页面组件中添加 JSON-LD
export default async function BlogPost({ params }: Props) {

    '@context': 'https://schema.org',

      '@type': 'Person',
      name: blog.author,
    },
    datePublished: blog.date,
    image: blog.image,
  }

  return (
    <>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      {/* 现有内容 */}
    </>
  )
}
```

## 📝 总结

此项目结构良好，但存在几个性能反模式。通过实施上述优化，特别是围绕图片优化、懒加载、静态生成和合并重复逻辑，可以显著提高性能和用户体验。

**预期影响**:
- 页面加载速度提升 30-50%
- Lighthouse 性能分数提升 15-25 分
- 服务器负载减少 40-60%
- 更好的 SEO 排名
- 改善的开发体验和类型安全

建议从**图片优化**和**静态生成**开始，因为它们能带来最大的即时影响。
